# Test Plan for Gingles.co

## 1. Введение

Gingles.co — это интернет‑магазин, специализирующийся на продаже имбирных напитков (ginger shots) различных вкусов и объёмов.  
Цель тестирования — проверить корректность работы всех пользовательских сценариев, стабильность функционала, удобство интерфейса и отсутствие критических ошибок, влияющих на покупку товара.

### Цели тестирования:
- выявить дефекты в функциональности сайта  
- проверить корректность работы корзины и оформления заказа  
- убедиться, что сайт корректно работает на разных устройствах  
- подготовить автотесты для ключевых сценариев

## 3. Виды тестирования

### 3.1. Функциональное тестирование
- добавление товара в корзину  
- изменение количества  
- удаление товара  
- оформление заказа  
- работа форм  
- отображение цен  
- работа меню и навигации  

### 3.2. UI/UX тестирование
- корректность отображения элементов  
- адаптивность  
- удобство интерфейса  
- читабельность текста  
- логичность пользовательских потоков  

### 3.3. Кроссбраузерное тестирование
- Chrome  
- Firefox  
- Edge  
- Safari (мобильный)  

### 3.4. Мобильное тестирование
- iPhone (Safari)  
- Android (Chrome)  
- Маленькие экраны (320px)  
- Средние экраны (768px)  
- Большие экраны (1024px+)  

### 3.5. Тестирование производительности
- скорость загрузки страниц  
- ошибки в консоли  
- стабильность работы корзины  
- корректность загрузки изображений  

### 3.6. Тестирование безопасности (базовое)
- корректность HTTPS  
- отсутствие XSS в формах  
- отсутствие утечек данных  

### 3.7. Smoke‑тестирование
- сайт открывается  
- товары отображаются  
- корзина работает  
- оформление заказа доступно  

### 3.8. Регрессионное тестирование
Проводится после изменений или релизов.

## 4. Что НЕ входит в тестирование

- нагрузочное тестирование (1000+ пользователей)  
- тестирование backend‑инфраструктуры  
- тестирование интеграций с внешними системами  
- тестирование админ‑панели (если нет доступа)  
- тестирование email‑уведомлений на реальных почтовых сервисах

## 6. Риски

### 6.1. Технические риски
- ошибки пересчёта корзины  
- нестабильная работа checkout  
- ошибки загрузки изображений  
- проблемы адаптивности  
- ошибки валидации форм  

### 6.2. Бизнес‑риски
- невозможность оформить заказ → потеря продаж  
- некорректные цены → финансовые потери  
- неработающие кнопки “Přidat do košíku”  
- ошибки в программе лояльности  

### 6.3. Риски тестирования
- тестирование на проде  
- отсутствие логов  
- изменения на сайте без уведомления
